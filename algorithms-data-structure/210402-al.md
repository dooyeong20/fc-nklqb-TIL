# 탐색

Search Algorithms

- 원하는 조건에 맞는 자료를 찾는 것을 탐색 알고리즘이라고 한다
- 자료가 정렬되어있지 않다면, 이분 탐색이 불가능하다.

- 미분가능하고 연속적이면 수치해석(?)

## 선형 탐색

- 순차 탐색(Sequential Search)
- 순서대로 하나씩 비교 -> O(n)

## 이분 탐색

- 이진 탐색
- 자료가 정렬되어 있을 때 적용 가능 (요소 중복 가능)
- 탐색 범위를 절반씩 줄여가며 탐색 -> O(logn)
- bst의 best 케이스라고도 볼 수 있음
- bst는 삽입할 때 logN, 검색할 때 logN이 걸린다
  - 자료가 추가가 되면서 자동적으로 정렬이 되기 때문에(logN) 좋다
- 리스트에는 자료를 추가해 이분탐색하려면 O(nlogn)이 걸리기 때문에 데이터 추가나 삭제 등을 고려하지는 않고 탐색용이다

# 해시

넓은 범위의 값이 주어졌을 때 특정 범위(값)으로 좁힌 값을 도출  
모듈러 연산(%)도 간단한 해시 값에 해당한다

- 해시, 해시 값
- 동일 입력에 대해 동일 출력을 보장

  - 다른 입력에 대해서도 동일한 출력이 발생할 수 있음(충돌)

- 해시 값의 범위를 좁히는 것과 해시 충돌이 일어나지 않도록 하는 것은 반비례한다. (비둘기 집의 원리(?))
- 임의 접근이 가능해진다

## 해시 성능에 영향을 주는 요소들(trade off 관계에 있다)

1. 해시함수의 동작 시간(해시를 계산하는데 걸리는 시간)
2. 해시 충돌의 발생 확률
3. 해시 값의 범위 (부작용이 없다면 범위가 작을수록 좋다)

## 해시 셋(set)

파이썬에서 set으로 바꿔주면 in 연산을 통해 O(1)의 시간복잡도로 데이터의 유무를 파악할 수 있다.

## 해시 충돌

- Bucket 에 담는 과정에서도(index 계산한다 등등) 해시 충돌이 일어날 수 있다
- 일반적으로 자료 저장 인덱스는 bucekt 사이즈로 모듈러 연산으로 구한다
